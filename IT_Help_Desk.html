Ticket Management System Project

This project implements a simple ticket management system using Python, Pandas, SQLAlchemy, and SQLite. It allows for logging, viewing, resolving, and analyzing tickets. The data is stored in a SQLite database, and the system is interactive through a command-line interface.

Project Description

The system is designed to handle helpdesk tickets, each with a CaseID, ActivityID, and CompleteTimestamp. The core functionalities include logging new tickets, viewing the status of a specific ticket, resolving tickets, and generating basic analysis on ticket activity. It utilizes Pandas for data handling, SQLAlchemy for database interaction, and SQLite as the database backend.

Key Features and Functionalities

  • Database Creation: A SQLite database is created, and initial ticket data is loaded from a CSV file.
  • Ticket Logging: Users can log new tickets by providing the Case ID, Activity ID, and timestamp.
  • Ticket Viewing: Users can view the details of a ticket using its Case ID.
  • Ticket Resolution: Users can mark a ticket as resolved by updating its Activity ID.
  • Ticket Analysis: The system can provide a count of tickets based on their Activity ID, providing insights into the current load.
  • Interactive Command-Line Interface: A menu-driven interface guides users through system functionalities.
