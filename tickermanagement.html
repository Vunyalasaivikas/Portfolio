Ticket Management System

Project Overview

This project implements a basic Ticket Management System using Python, Pandas, and SQLite. The system allows users to log new tickets, view ticket status, resolve tickets, and analyze ticket data. It interacts with an SQLite database to store and retrieve ticket information.

Core Functionalities

  • Database Setup:
    • Uses `sqlalchemy` to connect to and manage an SQLite database.
  • Data Loading and Storage:
    • Reads ticket data from a CSV file (`helpdesk.csv`) using pandas.
    • Loads the data into the SQL database.
  • Ticket Management:
    • Implements a function to log new tickets with `CaseID`, `ActivityID`, and `CompleteTimestamp` into the database.
    • Allows users to view tickets by their `CaseID`.
    • Implements a function to update the `ActivityID` of a ticket to 6, marking it as resolved.
  • Data Analysis:
    • Provides functionality to perform analysis to count number of ticket by `ActivityID`.
  • Interactive Interface:
    • A command-line menu interface provides options for logging, viewing, resolving, analyzing tickets and exiting the system

Technical Details

  • Programming Language: Python 3.11.
  • Libraries/Modules Used:
    • pandas: Used for data manipulation and analysis.
    • sqlalchemy: Used for database connection and management.
    • sqlite: Used for database storage
  • Data Storage: CSV (`helpdesk.csv`), SQLite database.
  • Output: Console output providing ticket information and analysis.

How to Use

  1. Prepare the Data:

    Ensure the CSV data file (`helpdesk.csv`) is in the same directory as your Python script or provide the correct file path.

  2. Run the Script:

    Execute the Python script or Jupyter Notebook containing the provided code.

  3. Interact with Menu:

    A menu will be displayed with options to log a ticket, view ticket status, resolve a ticket, analyze tickets, or exit. Select an option by entering the corresponding number.

  4. Follow the prompts:

    Follow the prompts to input required information, such as Case ID, Activity ID, and timestamps, if prompted.

  5. Review the Output:

    View the ticket information and analysis on the console.
