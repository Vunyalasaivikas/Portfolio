Ticket Management System

This project implements a basic Ticket Management System using Python, Pandas, and SQLite. The system allows users to log new tickets, view ticket status, resolve tickets, and analyze ticket data. It interacts with an SQLite database to store and retrieve ticket information.

Project Description

The Ticket Management System is designed to handle a basic workflow for tracking and managing tickets. It utilizes an SQLite database to persist ticket data, including Case IDs, Activity IDs, and Timestamps. The core functionality includes logging new tickets, viewing ticket details based on Case ID, resolving existing tickets by updating their Activity ID, and performing basic analysis by grouping tickets by Activity ID to count tickets.

Key steps and Features

  • Database Setup: Uses `sqlalchemy` to connect to and manage an SQLite database.
  • Data Loading: Reads ticket data from a CSV file (`helpdesk.csv`) using pandas.
  • Data Storage: Loads the data into the SQL database.
  • Logging Tickets: Implements a function to log new tickets with `CaseID`, `ActivityID`, and `CompleteTimestamp` into the database.
  • Viewing Tickets: Allows users to view tickets by their `CaseID`.
  • Resolving Tickets: Implements a function to update the `ActivityID` of a ticket to 6, marking it as resolved.
  • Ticket Analysis: Provides functionality to perform analysis to count number of ticket by `ActivityID`.
  • Interactive Interface: A command-line menu interface provides options for logging, viewing, resolving, analyzing tickets and exiting the system

How to Use

To interact with the Ticket Management System, follow these steps:

  1. Run the Python script containing the provided code.
  2. A menu will be displayed with options to log a ticket, view ticket status, resolve a ticket, analyze tickets, or exit.
  3. Select an option by entering the corresponding number.
  4. Follow the prompts to input required information, such as Case ID, Activity ID, and timestamps, if prompted.
  5. View the ticket information and analysis on the console.

Where It's Useful

This Ticket Management System can be useful in various sectors and applications:

  • Customer Support: Tracking and managing customer support tickets.
  • IT Helpdesk: Managing and resolving IT-related issues.
  • Project Management: Tracking project tasks as tickets.
  • Issue Tracking: Managing bugs or feature requests in software development.
  • Service Management: Tracking service requests and resolving incidents.
  • Small Teams: Ideal for small teams or organizations that need basic ticket management without complex features.
